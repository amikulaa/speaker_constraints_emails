# Speaker Constraints Collection System

## Overview
This system handles outreach to event speakers to collect scheduling constraints and availability.

Instead of relying on manual emails and scattered responses, it sends structured emails to speakers and organizes their responses in a centralized system that staff can use during event planning.

## Features
- Sends emails to speakers requesting availability and constraints
- Collects and stores responses in a structured format
- Provides a centralized view of speaker availability for event planning
- Reduces back-and-forth communication and missed information

## Tech Stack
- PHP
- JavaScript
- SQL
- WordPress-based environment

## Key Contributions
- Built a system to replace mailmerge outreach and manual tracking of speaker constraints
- Standardized how availability information is collected and stored
- Reduced scheduling issues caused by missing or inconsistent responses
- Improved coordination for staff managing event logistics

## Notes
Designed around real event planning workflows, where collecting and organizing constraints quickly and reliably is critical.
